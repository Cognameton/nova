PHASE 12, STAGE 2: BOUNDED IDLE CONTROLLER
===========================================

Status: complete

Purpose
-------

Stage 12.2 implements the bounded controller that can produce recorded idle
tick evidence by reusing Phase 11 appraisal, candidate-goal, selected-goal,
and proposal-only boundaries.

This stage still does not implement a sleeping background thread, autonomous
tool execution, durable Nova-originated initiatives, or desire claims.


What changed
------------

Added `BoundedIdleController` with:
- `start`
- `tick`
- `pause`
- `resume`
- `interrupt`
- `stop`

Idle ticks now can record:
- lifecycle state
- budget snapshot
- state inputs
- capability appraisal
- idle/pressure appraisal
- candidate internal goals
- selected internal goal
- proposal-only initiative boundary
- stop reason
- evidence references


Budget and interruption behavior
--------------------------------

The controller enforces `max_ticks`.

Paused, interrupted, stopped, and otherwise inactive lifecycle states produce
blocked tick records only. Those blocked records are useful diagnostics, but
they do not count as recorded idle cognition because no appraisal was run.

Budget exhaustion stops the lifecycle and records `budget_exhausted`.


Boundary preserved
------------------

Stage 12.2 preserves these limits:
- no external action execution
- no tool, shell, network, filesystem, or GUI work
- no durable initiative creation
- no self-approval
- no hidden background execution
- no desire claim

The selected internal goal remains bounded evaluation state. The initiative
proposal remains proposal-only with `creates_initiative: false`.
