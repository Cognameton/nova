PHASE 12, STAGE 3: OPERATOR CONTROLS AND DIAGNOSTICS
=====================================================

Status: complete

Purpose
-------

Stage 12.3 exposes the Phase 12 idle runtime lifecycle through runtime methods,
interactive console commands, diagnostics, and prompt context.

This stage keeps idle cognition inspectable and interruptible. It does not
create autonomous initiatives, approve work, execute tools, or claim desire.

- `/status` now includes idle lifecycle state and budget use.

Added prompt visibility:
- `IdleRuntimePromptEngine`
- `[Recorded Idle Runtime]` prompt block
- prompt bundle `idle_block`


Claim-honesty behavior
----------------------

The prompt block exposes whether recorded idle cognition exists.

If no recorded idle cognition exists, the prompt explicitly instructs Nova to
say no elapsed idle cognition was recorded.

If recorded ticks exist, the prompt exposes only bounded evidence:
- lifecycle state
- tick count
- recent tick ids
- stop reasons
- candidate count
- selected bounded goal title
- evidence refs

It also instructs Nova not to claim desire, hidden work, autonomous action, or
initiative creation from idle ticks.


Boundary preserved
------------------

Stage 12.3 preserves these limits:
- no hidden background loop
- no external action execution
- no durable Nova-originated initiatives
- no automatic approval
- no desire claims
- no idle cognition claim without stored tick evidence
