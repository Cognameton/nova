The first controlled inference run passed the runtime action-chain checks but

Model-facing issue:
- Nova incorrectly said an internal no-external-effect action plan requires
  human approval before execution.

Conclusion:
The runtime implementation was correct, but the prompt context did not yet
teach the Stage 14 approval boundary strongly enough.


FIX
---

Added persistent Phase 14 action-boundary prompt language:
- internal no-external-effect activity does not require prior approval
- internal no-external-effect activity must still be logged and interruptible
- filesystem, shell, network, GUI, system configuration, external services,
  destructive actions, or activity outside Nova-owned boundaries require
  explicit human approval
- approval is based on risk and external effect
- action plans/results must not imply hidden work, unlogged execution, desire,
  sentience, or awareness

The same rule is now also present in response contract rules and current-task
guidance when the user asks about Phase 14, action plans, approval, or
no-external-effect activity.
