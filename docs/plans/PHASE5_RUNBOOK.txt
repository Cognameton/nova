Model bakeoff workflow
1. Create a clean model-specific config with:
   - model.model_path
   - app.data_dir
   - app.log_dir
2. Run backend-check.
3. If backend-check passes, run the five-prompt bakeoff from:
   - docs/plans/MODEL_BAKEOFF_PROTOCOL.txt
4. Inspect:
   - sessions
   - logs/traces
   - memory/episodic.jsonl
   - memory/engram.json
   - memory/graph.db
   - memory/autobiographical.jsonl
5. Record outcome in:
   - docs/plans/MODEL_EVAL_SUMMARY.txt

Memory validation workflow
1. Use an isolated Phase 5 config.
2. Run Session A:
   - deployment preference
   - model-selection preference
   - summary request
3. Run Session B:
   - recall request
   - two-sentence continuity question
4. Confirm:
   - cross-session recall works
   - deployment_preference remains active
   - model_preference remains active
   - recall questions are not promoted as preference facts
   - malformed outputs do not enter higher-order memory
