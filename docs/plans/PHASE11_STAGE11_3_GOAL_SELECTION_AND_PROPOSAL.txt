PHASE 11, STAGE 3: BOUNDED GOAL SELECTION AND INITIATIVE PROPOSAL
==================================================================

Status: complete

Purpose
-------

Stage 11.3 adds bounded selection over provisional candidate internal goals
and converts a selected goal into a proposal-only initiative proposal.

This stage does not create `InitiativeRecord` entries.
This stage does not approve initiatives.
This stage does not execute tools or actions.
This stage does not claim desire.


What changed
------------

Added `SelectedInternalGoal`:
- selected candidate id
- selection reason
- priority score
- approval/proposal requirements
- blocked/rejection state
- evidence references

Added `InternalGoalInitiativeProposal`:
- proposal id
- candidate id
- title and goal text
- proposal-only status
- approval-required flag
- `creates_initiative: false`
- empty `initiative_id`

Added engines:
- `InternalGoalSelectionEngine`
- `InternalGoalInitiativeProposalEngine`
- `SelectedGoalPromptEngine`

Added runtime integration:
- selected internal goal is included in turn notes
- proposal-only initiative proposal is included in turn notes
- both are included in traces
- selected-goal prompt block is rendered when selection succeeds
- initiative state is not mutated


Selection rules
---------------

Selection only considers candidates that are:
- provisional
- approval-required
- selection-eligible
- not rejected
- not dependent on blocked capability surfaces

Current ranking order:
- `initiative_resumption_candidate`
- `bounded_skill_learning`
- `uncertainty_resolution`
- `capability_clarification`


Boundary
--------

The selected goal is a bounded evaluation output, not desire.

The proposal is a proposal-only record, not an initiative. Initiative creation
still belongs to the explicit Phase 9 initiative path and requires approval.
