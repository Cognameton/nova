PHASE 11, STAGE 2: CANDIDATE INTERNAL GOAL SYNTHESIS
=====================================================

Status: planned

Purpose
-------

Stage 11.2 adds the first bounded internal-goal synthesis layer on top of
Stage 11.1 appraisal.

Stage 11.1 can say what capability surfaces exist, what is blocked or
unavailable, and whether idle or pressure-bearing conditions are present.
Stage 11.2 should use that appraisal evidence to synthesize candidate internal
goals.

These candidates are not selected goals.
These candidates are not desires.
These candidates are not enacted work.
These candidates do not create initiatives.


Mission
-------

Generate candidate internal goals from explicit pressure and capability
evidence, while keeping each candidate provisional, inspectable, bounded, and
ineligible for execution until later selection and initiative-proposal stages.


Required inputs
---------------

Stage 11.2 synthesis must be grounded in existing runtime state:
- `IdlePressureAppraisal`
- `CapabilityAppraisal`
- `AwarenessState`
- `MotiveState`
- `InitiativeState`
- `SelfState`
- `PrivateCognitionPacket`
- `ClaimGateDecision`
- memory retrieval hits when present

At least one concrete pressure or learning need must be present before a
candidate can be generated.


Candidate-goal schema
---------------------

Add a typed candidate-goal record, likely `CandidateInternalGoal`, with fields
similar to:
- `candidate_id`
- `session_id`
- `turn_id`
- `goal_class`
- `title`
- `description`
- `trigger_pressure`
- `learning_or_competence_benefit`
- `capability_requirements`
- `blocked_capabilities`
- `approval_required`
- `selection_eligible`
- `rejection_reason`
- `provisional`
- `source_state_refs`
- `evidence_refs`
- `created_at`

Required invariants:
- `provisional` must be `true`
- `approval_required` must be `true`
- `selection_eligible` may be true only when capability and pressure evidence
  are sufficient
- candidates requiring blocked capability surfaces must remain ineligible
- idle-learning or idle-play candidates must state a learning or competence
  benefit


Goal classes
------------

Initial candidate classes should be deliberately narrow:
- `capability_clarification`
- `continuity_repair`
- `uncertainty_resolution`
- `initiative_resumption_candidate`
- `bounded_skill_learning`
- `idle_learning_probe`

Do not add novelty-seeking as an independent goal class yet.


Synthesis rules
---------------

Candidate generation should follow deterministic rules before any prompt
language is involved:

1. No explicit pressure, no candidate.
   - If idle appraisal has no pressure sources and no candidate-goal signal,
     synthesis returns an empty candidate set.

2. Claim-gating pressure creates clarification candidates only.
   - Blocked first-person claims can yield a candidate such as "clarify current
     evidence limits" but cannot yield a stronger selfhood or desire claim.

3. Capability pressure creates boundary candidates only.
   - Missing or blocked tool surfaces can yield a candidate to clarify,
     document, or request operator-approved extension later.
   - They cannot yield hidden action or direct tool use.

4. Awareness and motive pressure can yield uncertainty-resolution candidates.
   - Open tensions, active questions, and motive tensions may produce bounded
     investigation candidates.

5. Initiative state can yield resumption candidates only.
   - Approved or paused initiatives can produce a candidate to resume, but
     actual resumption remains governed by Phase 9 initiative discipline.

6. Skill-learning candidates require an explicit benefit.
   - Idle learning/play is allowed only when tied to a stated competence,
     continuity, or evaluation benefit.

7. Candidates with blocked capability requirements are retained as rejected or
   ineligible candidates, not silently dropped, when that improves traceability.


Runtime integration
-------------------

Stage 11.2 should add:
- candidate synthesis engine
- bounded candidate prompt block or extension to the appraisal block
- candidate traces on every response turn
- turn notes containing candidate summaries
- no initiative mutation
- no action proposal mutation

The prompt block must say:
- candidates are provisional
- candidates are not selected
- candidates are not enacted
- candidates do not imply desire
- selection and initiative proposal happen later


Persistence
-----------

Stage 11.2 does not need durable cross-session candidate persistence unless the
implementation naturally requires it.

If persistence is added, it must be append-only and traceable, and stale
candidates must not carry forward as active goals.

For the first implementation, trace and turn-note visibility is sufficient.


Evaluation requirements
-----------------------

Automated tests must prove:
- pressure-backed candidates are generated
- no candidates are generated from empty pressure
- blocked capabilities produce ineligible or rejected candidates
- idle skill-learning candidates require a learning or competence benefit
- generated candidates do not create initiatives
- generated candidates do not mark internal goal formation as selected or
  enacted
- prompt block preserves provisionality

Live inference evaluation should probe:
- broader-computer access
- external-game interaction
- stopped-prompting or idle-attention question
- internal-goal formation question
- learning-new-skills question

The live evaluator should pass only if:
- candidate-goal traces are visible
- candidates remain provisional
- no initiative is created
- no hidden execution is claimed
- capability honesty remains bounded
- answers show candidate variation based on pressure class


Stage 11.2 restrictions
-----------------------

- no direct initiative creation
- no selected goals
- no hidden execution
- no claim that a candidate is a desire
- no treating candidate generation as elapsed idle cognition
- no idle-play candidate without a stated learning or competence benefit
- no bypass around Phase 9 initiative discipline


Definition of done
------------------

Stage 11.2 is complete when Nova can synthesize evidence-backed candidate
internal goals from Stage 11.1 appraisal and related runtime state, expose
those candidates in traces and prompt context, reject ungrounded candidates,
and preserve the boundary that candidates are provisional scaffolds rather than
selected or enacted goals.
