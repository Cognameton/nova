Nova 2.0 Phase 1 Plan

Mission
Nova 2.0 Phase 1 establishes a stable, text-only local runtime that can host a candidate GGUF model under a fixed identity, prompt, memory, and output contract. Phase 1 is not about agentic behavior, voice, or model bakeoff. It is about creating a disciplined experimental harness for continuity-oriented local inference.

Phase 1 Objectives
1. Create one canonical text runtime for Nova.
2. Define and enforce a no-<think> output contract.
3. Persist persona state and mutable self-state separately.
4. Establish baseline memory channels and retrieval plumbing.
5. Produce full session and trace logging for forensic inspection.
6. Add basic continuity probes.
7. End with a runnable CLI and smoke-test coverage.

Architectural Principles
- Local inference only.
- One canonical runtime path.
- Model artifacts live outside the repo.
- Prompt composition is deterministic and inspectable.
- Memory channels have distinct responsibilities.
- Invalid generations do not poison long-term memory.
- Logging and traces are first-class, not optional extras.
- Architecture cleanliness takes priority over feature sprawl.

Prompt Contract
Nova responses must:
- respond directly
- avoid visible chain-of-thought
- never emit <think> tags
- avoid prompt/section echoing
- avoid leaking internal planning language

If validation fails:
1. mark the output invalid
2. retry with a concise repair instruction
3. fall back if necessary
4. never write invalid generations into long-term memory

Prompt Composition Order
1. Persona
2. Self-State
3. Memory blocks
4. Recent conversation
5. User input
6. Response rules

Phase 1 Memory Roles
Episodic memory
- append-oriented event history
- raw user and assistant turns

Engram memory
- associative pattern memory
- recurring phrasing, motifs, identity-relevant wording

Graph memory
- structured facts and relations
- self, user, preference, project, runtime facts

Autobiographical memory
- privileged continuity-bearing notes
- identity-relevant summaries and self-model continuity

Semantic memory
- intentionally deferred as a true layer until Phase 2

Session Lifecycle
1. load config
2. load persona state
3. load self-state
4. initialize backend
5. start or resume session
6. retrieve memory hits
7. compose prompt
8. generate output
9. validate output
10. retry if needed
11. persist turn
12. write memory events
13. log trace
14. log probes if enabled

Phase 1 Testing
Smoke coverage should verify:
- config override merging
- prompt composition
- output validation and retry policy
- session persistence round-trip
- episodic retrieval
- one full runtime turn with a fake backend

Phase 1 Exit Criteria
Phase 1 is complete when:
- Nova runs through one canonical text-only runtime
- one local GGUF model can be loaded and queried reproducibly
- output validation blocks or retries visible reasoning leakage
- persona and self-state persist across sessions
- sessions and traces are logged every turn
- baseline memory channels are active
- continuity probes exist and run
- the CLI is usable
- smoke tests pass
