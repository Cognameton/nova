PHASE 6 STAGE 6.4 VALIDATION
============================

Stage:
- Stage 6.4: Private Cognition Layer

Mission:
- Add a bounded pre-response cognition layer that helps Nova evaluate
  continuity and response quality without exposing hidden reasoning or
  multiplying latency.


WHAT CHANGED
------------

1. A deterministic private cognition packet now exists on the pre-response
   path.
- The runtime now builds an inspectable `PrivateCognitionPacket` before
  generation.
- The packet is structured rather than freeform and includes:
  - trigger
  - continuity_risk
  - memory_conflict
  - response_mode
  - revise_needed
  - uncertainty_flag
  - pass budget usage
  - governing memory ids

2. The cognition layer is gated rather than universal.
- Cognition does not run on ordinary light-format or declarative preference
  turns.
- Cognition does run on continuity-sensitive recall/conflict turns.
- This keeps Stage 6.4 bounded instead of turning every response into a hidden
  multi-pass workflow.

3. The pass budget and revision ceiling are explicit.
- New config section:
  - `cognition.enabled`
  - `cognition.pass_budget`
  - `cognition.revision_ceiling`
- Default path uses:
  - pass budget: `1`
  - revision ceiling: `1`
- No extra hidden generation pass is introduced.
- The same packet is reused across any normal output retries.

4. The packet is inspectable in traces and prompt bundles.
- Trace records now persist the private cognition packet.
- Prompt bundles now include a structured `[Private Cognition]` block only when
  the gate opens.
- The block contains bounded instructions such as preferring active over
  archived/historical continuity memory.

- no `<think>` leakage appeared in the final answer
