PHASE 13, STAGE 1: AUTONOMOUS INITIATIVE ORIGIN AND APPROVAL SCHEMA
====================================================================

Status: complete

Purpose
-------

Stage 13.1 adds the durable data contracts needed before Nova-originated draft
initiatives can be materialized from idle/internal-goal evidence.

This stage does not create Nova-originated initiative records automatically.
It only extends the schema and normalization rules so later stages can do so
without changing the meaning of existing Phase 9 initiative lifecycle states.


What changed
------------

Added initiative origin taxonomy:
- `user`
- `nova`
- `system`
- `runtime`
- `cli`

Added initiative approval-state taxonomy:
- `draft`
- `awaiting_user_approval`
- `approved`
- `rejected`
- `paused`
- `abandoned`
- `complete`
- `not_required`

Extended `InitiativeRecord` with:
- `origin_type`
- `approval_state`
- `source_idle_tick_id`
- `source_candidate_id`
- `source_proposal_id`
- `rationale`
- `proposed_next_step`
- `stop_condition`
- `autonomous`

Added normalization helpers:
- `normalize_initiative_origin_type`
- `normalize_initiative_approval_state`


Compatibility behavior
----------------------

Older initiative records continue to load.

When older records do not include the new fields:
- `origin_type` derives from `source`
- `approval_state` derives from existing lifecycle `status`
- autonomous provenance fields default empty
- `autonomous` defaults true only for `origin_type: nova`

Existing lifecycle states remain intact:
- `pending`
- `approved`
- `active`
- `paused`
- `completed`
- `blocked`
- `abandoned`


Boundary preserved
------------------

Stage 13.1 preserves these limits:
- no Nova-originated record creation yet
- no automatic approval
- no active autonomous initiatives
- no action execution
- no desire claims

The new schema can represent a Nova-originated draft, but later stages must
decide when a proposal is eligible to create one.
